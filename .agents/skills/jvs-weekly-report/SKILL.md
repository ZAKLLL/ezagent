# JVS 周报生成

结合用户提供的工作说明/会议纪要与当期 OKR，生成格式统一的周报。

## 输入

用户会提供以下内容之一（或两者）：
- **本周工作说明 / 会议纪要**：文字描述的工作内容、跨团队协作、卡点
- **时间范围**：如「本周」「4月25-30日」「W18」（若未提供则默认当前自然周）

## 执行流程

### Step 1：确定时间范围

若用户未明确说明周次和日期，按如下规则推算：
- 当前日期 → 推算本周一到周五
- 若今天是周末，默认取上周
- 将时间范围转换为 `W{周次}_{起始月日}_{结束月日}` 格式，例：`W18_0425_0430`

### Step 2：读取当期 OKR

读取 `work/work_report/` 下当前季度的 OKR 文件（如 `okr_template.md` 复制出的 `2026_Q2.md`），提取各 Objective 和 KR 的核心目标。若用户尚未维护 OKR 文件，跳过此步。

OKR 的作用是**帮助归类和描述**，而不是在正文里贴标签：
- 理解每项工作属于哪个方向，生成周报时按 OKR 所定义的业务维度自然聚合
- 若某项工作推进了某个 KR，在对应节的描述里体现（用业务语言，不写"KR1"）
- 若本周有 OKR 相关的卡点或风险，在下周计划里反映

### Step 3：整理内容

将用户提供的工作说明按主题聚合，原则：
- 以用户语言描述工作内容，具体、可核查
- 保留跨团队协作、卡点、风险等信息
- 若信息不足，向用户追问关键工作事项，而不是编造内容

### Step 4：生成周报

按以下模板生成，**正文不出现 OKR/KR 字样**，OKR 只作头部静默链接：

```markdown
# 周汇报 W{N}（{起始日期} — {结束日期}）

> 关联 OKR：[2026 Q{N} OKR](../2026_Q{N}.md)

---

## 本周工作

### {主题一}

- 具体工作内容
- ...

### {主题二}

- ...

---

## 下周计划

{根据用户说明填写，若无则留"待补充"}

---

## 风险与需要协助

{跨团队协作、卡点、风险，若无则留"暂无"}
```

**格式规范：**
- 按工作主题分节，不按 OKR/KR 分节
- 每节用自然语言描述，不堆砌术语
- 周报整体控制在 40-80 行，不要过长

### Step 5：保存并更新索引

1. 文件路径：`work/work_report/weekly/{year}/W{N}_{起始月日}_{结束月日}.md`
2. 同步更新 `work/work_report/README.md` 的周报索引表
3. 按 [[jvs-memory]] 或用户要求，提交 Git（本地仓库无远程时，`git add` + `git commit` 即可，不做 `git push`）

## 注意事项

- 若对应周的文件已存在，询问用户是否覆盖
- 下周计划若用户已明确提供，原文保留；若只是隐含在工作说明里，提炼出来
- 不创建 OKR 文件，OKR 由用户参照 `work/work_report/okr_template.md` 单独维护
