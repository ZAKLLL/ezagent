# JVS 知识笔记系统

此技能指导 AI Agent 使用 EzAgent 的知识笔记库，管理和组织日常工作中积累的行业认知、方法论与个人成长记录。

## 快速参考

### 笔记位置

```
notes/
├── README.md              # 目录说明
├── OBSIDIAN_NOTES.md       # 可选：用 Obsidian 管理笔记
├── 00_Archive/             # 归档内容（过期资料、历史记录）
├── 01_行业知识/            # 所在行业的知识、趋势、术语
├── 02_工作方法/            # 项目管理、时间管理、汇报技巧等方法论
├── 03_会议与沟通/          # 会议纪要模板、沟通技巧、谈判要点
├── 04_职业发展/            # 职业规划、技能提升、绩效复盘
├── 05_工具与效率/          # 常用办公工具、效率技巧
└── 06_个人成长/            # 读书笔记、反思、习惯养成
```

分类不够用时可以直接新建目录，不必拘泥于以上列表。

## 核心操作

### 添加笔记
```bash
# 在合适的分类目录下新建一篇笔记
NOTES_DIR="notes/02_工作方法"
NOTES_FILE="$NOTES_DIR/笔记标题.md"
mkdir -p "$NOTES_DIR"
```
用 Write/Edit 工具按下方标准格式写入内容。

### 搜索笔记
```bash
# 全局搜索关键词
grep -rn "关键词" notes/

# 限定分类搜索
grep -rn "关键词" notes/04_职业发展/
```

### 编辑笔记
用 Read 工具读取现有笔记，再用 Edit 工具修改内容，避免整篇覆盖。

## 标准笔记格式

```markdown
# [笔记标题]

## 元信息
- **分类**: [如 02_工作方法]
- **创建时间**: YYYY-MM-DD
- **标签**: #标签1 #标签2

## 核心内容
- 要点 1
- 要点 2

## 参考资源
- [资源链接](URL)

## 相关笔记
- [[相关笔记标题]]

## 待办/下一步
- [ ] 待办事项 1

---
*AI Agent: EzAgent*
```

## 笔记类型

- **概念笔记**：术语、理论、行业知识的定义与解释
- **方法笔记**：汇报技巧、会议方法、时间管理等可复用的做事方式
- **复盘笔记**：项目/阶段/绩效复盘，记录得失与下一步行动
- **资源笔记**：工具、书籍、课程等外部资源的推荐与要点摘录

## Agent 笔记工作流程

1. **识别需求**：用户提到"记一下""总结一下""帮我整理笔记"等诉求时触发
2. **确定分类**：参考上方分类列表，选一个最贴近的目录；找不到合适分类就新建
3. **搜索是否已有相关笔记**：`grep -rn "关键词" notes/`，避免重复记录，优先在已有笔记上追加
4. **按标准格式写入**，补全元信息和标签
5. **必要时联动记忆系统**：如果这条信息也是近期工作上下文（而不是长期知识），改用 `jvs-memory` 技能记录，而不是笔记

## 笔记最佳实践

- **及时记录**：读到、听到、想到时立即记，避免事后遗忘细节
- **具体明确**：记结论和可执行要点，少记流水账
- **善用标签与双链**：`#标签` 便于检索，`[[笔记名]]` 建立笔记之间的关联
- **定期整理**：每月回顾一次笔记分类，合并重复内容、归档过时内容到 `00_Archive/`

## 与记忆系统的边界

- **笔记（notes/）**：沉淀性的知识和方法论，长期有效，按主题分类
- **记忆（memory/）**：某个时间点的工作上下文和待办，按日期存档，参见 [[jvs-memory]] 技能

## 可选：Obsidian 管理

`notes/` 目录本身是纯 Markdown 文件，可以直接作为 Obsidian vault 打开使用双向链接和图谱视图，详见 `notes/OBSIDIAN_NOTES.md`；不使用 Obsidian 完全不影响正常使用。
