# JVS 更新文档

此技能指导 AI Agents 更新 EzAgent 仓库的核心文档，确保文档与实际代码保持同步。

## 目标文档

### 核心文档
- **README.md** - 项目总览和快速开始指南
- **CLAUDE.md** - 面向 Claude Code 的行为准则与项目说明（Claude Code 会自动读取）
- **AGENTS.md** - 数字分身人格定义和行为准则

### 参考文档
- 各 `.agents/skills/*/SKILL.md` 的技能说明
- 脚本文件的头部注释

## 更新触发条件

在以下情况需要更新文档：

### 系统升级
- 添加新的核心功能
- 修改现有功能的行为
- 删除不再支持的功能
- 架构重大调整

### 文档不一致
- README 描述的功能与实际代码不符
- AGENTS.md 中的行为准则需要调整
- 示例命令无法正常工作
- 文档中提到已删除的文件或目录

### 定期维护
- 发现过时信息
- 添加新的使用示例
- 优化文档结构

## 更新流程

### 1. 分析变更
```bash
# 查看最近的提交
git log --oneline -10

# 查看具体变更
git diff HEAD~5 HEAD

# 查看新增的文件
git diff HEAD~5 HEAD --name-only --diff-filter=A

# 查看删除的文件
git diff HEAD~5 HEAD --name-only --diff-filter=D
```

### 2. 检查当前文档状态
```bash
# 读取当前文档
cat README.md
cat AGENTS.md

# 检查文档中提到的文件是否存在
ls -la $(grep -oP '(?<=\`)[a-zA-Z0-9_\-./]+(?=\`)' README.md)

# 检查脚本是否存在
ls -la *.sh
```

### 3. 识别需要更新的内容

#### README.md 更新点
- **新增功能**：在"核心特性"中添加
- **新增脚本**：在"快速开始"或"文档"中添加使用说明
- **目录结构**：更新目录结构说明
- **示例命令**：验证并更新命令示例

#### CLAUDE.md / AGENTS.md 更新点
- **新增技能**：在"可用技能"中添加
- **行为准则**：调整或新增行为准则
- **工具使用**：更新可用工具列表

### 4. 更新文档

#### 读取文档
```bash
# 使用 Read 工具读取
# README.md
# AGENTS.md
```

#### 识别过时内容
- 功能描述与实际不符
- 提到已删除的文件
- 命令示例无效
- 版本信息过时

#### 更新内容
保持以下一致性：
- 命名风格统一
- 格式一致（标题、列表、代码块）
- 链接有效
- 示例可运行

### 5. 验证更新
```bash
# 检查 Markdown 语法
# （如果有 markdown lint 工具）

# 验证链接
# （如果有链接检查工具）

# 检查示例命令
./example-command-from-readme.sh
```

### 6. 提交更改
```bash
git add README.md CLAUDE.md AGENTS.md
git commit -m "docs: 更新文档以反映最新变更"
```

## 常见更新场景

### 场景 1：添加新脚本

**变更**：新增 `auto_sync_repo.sh`

**README.md 更新**：
- 在"快速开始"部分添加使用说明
- 在"目录结构"中提及
- 如果是核心脚本，在根目录列出

**示例**：
```markdown
### 自动同步仓库

```bash
# 设置每天8点自动同步
0 8 * * * /path/to/auto_sync_repo.sh
```
```

### 场景 2：新增 Skill

**变更**：新增 `jvs-update-doc` skill

**AGENTS.md 更新**：
- 在"可用技能"中添加技能描述
- 更新技能触发条件

**示例**：
```markdown
### jvs-update-doc
- **用途**：更新项目文档
- **触发**：系统升级、功能变更、架构调整
- **权限**：可读取和编辑 .md 文件
```

### 场景 3：删除功能

**变更**：移除某个不再使用的脚本/技能

**README.md 更新**：
- 从"核心特性"/"快速开始"中删除对应说明
- 删除相关的命令示例

**CLAUDE.md / AGENTS.md 更新**：
- 从"可用技能"列表中移除
- 删除对应的行为准则条目

### 场景 4：目录结构调整

**变更**：新增或重命名某个顶层目录（如新增 `work/xxx/`）

**README.md 更新**：
- 更新目录结构说明，保持与实际 `ls` 结果一致
- 补充该目录的用途说明和示例

## 自动化检查

建议创建自动化脚本检查文档一致性：

```bash
#!/bin/bash
# check-docs.sh

# 检查 README 中提到的脚本是否存在
grep -oP '\./[a-zA-Z0-9_\-]+\.sh' README.md | while read script; do
    if [ ! -f "$script" ]; then
        echo "警告: README 提到但脚本不存在: $script"
    fi
done

# 检查 Agent 中提到的技能是否存在
grep -oP 'jvs-[a-z]+' CLAUDE.md AGENTS.md | sort -u | while read skill; do
    if [ ! -d ".agents/skills/$skill" ]; then
        echo "警告: 文档提到但 skill 不存在: $skill"
    fi
done
```

## 最佳实践

### 更新时机
- 在功能开发完成后立即更新
- 删除代码前删除相关文档
- 大版本升级时全面审查
- 定期（每月）检查文档准确性

### 更新原则
- **准确优先**：宁可少写也要准确
- **简洁明了**：避免冗余和重复
- **示例可用**：所有示例都应可运行
- **版本一致**：文档与代码版本同步
- **易于维护**：结构清晰，便于后续更新

### 写作风格
- 使用一致的术语
- 保持简洁，避免过度说明
- 代码块使用正确的语言标记
- 使用空行分隔不同章节
- 列表使用一致的格式

## 故障排除

### 文档与代码不一致
1. 运行 `git diff` 查看近期变更
2. 对比文档与实际代码
3. 更新文档以反映最新状态
4. 验证更新后的准确性

### 示例命令失败
1. 检查命令语法
2. 验证脚本存在且有执行权限
3. 检查路径是否正确
4. 更新示例为可运行的命令

### 链接无效
1. 验证目标文件存在
2. 检查链接格式
3. 更新或删除无效链接

## 验证命令

### 完整验证流程
```bash
# 1. 检查文档中提到的所有脚本
for script in $(grep -oP '\./[a-zA-Z0-9_\-]+\.sh' README.md); do
    [ ! -f "$script" ] && echo "缺失: $script"
done

# 2. 检查文档中提到的所有目录
for dir in $(grep -oP '[a-zA-Z0-9_\-]+/' README.md | sort -u); do
    [ ! -d "$dir" ] && echo "缺失: $dir"
done

# 3. 检查技能
for skill in $(grep -oP 'jvs-[a-z]+' CLAUDE.md AGENTS.md | sort -u); do
    [ ! -d ".agents/skills/$skill" ] && echo "缺失: $skill"
done
```
