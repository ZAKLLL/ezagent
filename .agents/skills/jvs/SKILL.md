# JVS 快速入口

此技能让你在任何目录下快速进入 EzAgent 工作空间。

## 核心功能

当用户调用 `/jvs` 时，Agent 应：

1. **【关键】先捕获用户当前工作目录** - 在切换之前记录 `pwd`
2. **切换到 EzAgent 仓库目录**
3. **加载记忆上下文**
4. **准备接受用户任务**

## ⚠️ 重要：上下文保护

**当用户带有具体任务调用 /jvs 时（如同步文件、记一笔账），必须先捕获用户的当前目录，再切换！**

```
错误做法：
用户: /jvs 帮我把这份文档存进去
Agent: cd ~/ezagent  ← 直接切换，丢失用户上下文！
Agent: 你想存哪份文档？  ← 用户目录信息已丢失

正确做法：
用户: /jvs 帮我把这份文档存进去
Agent: USER_PWD=$(pwd)  ← 先记录用户当前目录
Agent: cd ~/ezagent
Agent: 准备处理来自 $USER_PWD 的文档
```

## 进入仓库后的标准动作

### Step 1：运行状态脚本

```bash
./agent_stats.sh
```

这会显示：
- 记忆、笔记、书签统计
- Git 状态

### Step 2：加载今日记忆

```bash
TODAY=$(date +%Y_%m_%d)
MEMORY_FILE="memory/mem_$TODAY.md"

if [ -f "$MEMORY_FILE" ]; then
    echo "今日记忆："
    cat "$MEMORY_FILE"
fi
```

## 典型使用场景

### 场景 1：同步文件到仓库

```
用户在 ~/Downloads 目录整理了一份会议纪要
用户: /jvs 帮我把这份纪要存进笔记库

Agent:
  1. 【先记录】捕获用户当前目录
  2. 切换到 ezagent 仓库
  3. 根据内容判断放到 notes/03_会议与沟通/ 还是 work/documents/
  4. 复制/整理内容，按需提交 Git
```

### 场景 2：记录记忆

```
用户: /jvs 记一下今天和老板对齐的几个要点

Agent:
  1. 切换到 ezagent 仓库
  2. 调用 ./memory_add.sh 写入记忆
  3. 确认记录完成
```

### 场景 3：查看状态

```
用户: /jvs

Agent:
  1. 切换到 ezagent 仓库
  2. 运行 ./agent_stats.sh
  3. 显示当前状态
  4. 等待用户指令
```

## 文件同步规则

**优先询问用户放到哪里**，若用户未指定则按以下规则粗判：

| 内容类型 | 默认位置 |
|---------|---------|
| 长期知识/方法论 | `notes/{对应分类}/` |
| 近期工作上下文/待办 | `memory/`（通过 `memory_add.sh`） |
| 会议纪要/简历等文档 | `work/documents/` |
| 常用链接 | `work/bookmarks/` |
| 周报/OKR/复盘 | `work/work_report/` |

不确定归属时，直接问用户，不要凭空猜测。

## 可用脚本

进入仓库后可使用：

| 脚本 | 用途 |
|------|------|
| `./agent_stats.sh` | 显示状态面板 |
| `./memory_add.sh` | 添加记忆 |
| `./memory_search.sh [关键词]` | 搜索记忆 |
| `./cron_manage.sh` / `./cron_query.sh` | 管理/查询定时任务 |

## Agent 行为准则

### 必须执行

1. **【最重要】上下文保护**：切换目录前先记录用户当前 `pwd`
2. **切换目录**：进入 EzAgent 仓库
3. **显示状态**：运行 `./agent_stats.sh` 展示当前上下文
4. **加载记忆**：读取今天的记忆文件

### 推荐执行

1. **主动询问**：用户需要什么帮助？
2. **提供建议**：基于记忆上下文建议相关操作

### 禁止执行

1. **【严重】未记录 pwd 就切换目录**：会丢失用户的当前上下文！
2. **跳过状态检查**：确保了解当前仓库状态

## 响应模板

```
📍 已进入 EzAgent

📊 当前状态：
- 记忆: {N} 条
- 笔记: {N} 篇
- 书签: {N} 条

📝 今日记忆预览：
{今日记忆摘要}

💬 需要我帮你做什么？
- 同步文件
- 记录记忆
- 查看状态
- 其他任务
```

## 故障排除

### 仓库目录不存在

```
检测到仓库目录不存在，请检查：
1. 仓库路径是否正确？
2. 是否需要先 clone/初始化？
```

### 脚本权限问题

```bash
chmod +x *.sh
```

## 与其他技能的协作

- **jvs-memory**：记录和搜索记忆
- **jvs-notes**：管理知识笔记
- **jvs-keep-clean**：清理仓库
- **jvs-update-doc**：更新文档
- **jvs-weekly-report**：生成周报
